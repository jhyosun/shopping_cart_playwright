# Playwright 활용 이커머스 웹 서비스 자동화 테스트
![image.png](images/image.JPG)

### 🔹 개요
[SauceDemo](https://www.saucedemo.com/) 데모 쇼핑몰을 대상으로 사용자 핵심 기능인 로그인, 상품 장바구니, 체크아웃 흐름에 대한 자동화 테스트를 수행했다.

### 🔹 목표 (수정 참고만)

- 로그인 기능의 정상 동작 여부를 자동으로 검증한다.
- 잘못된 계정 정보 입력 시 적절한 오류 메시지가 출력되는지 확인한다.
- 로그인 이후 로그아웃 기능이 정상 동작하는지 검증한다.
- 반복 가능한 테스트 환경을 구축하여 수동 테스트의 비효율을 줄인다.
- 실무에서 자주 사용하는 자동화 테스트 구조(POM, fixture, parameterize)를 학습하고 적용한다.

### 진행할 테스트 (정리중)

- 로그인 성공/실패
- 상품 리스트 확인
- 장바구니 담기
- 장바구니 삭제
- 주문(Checkout)



### 🔹 자동화 시나리오 설계

다음과 같은 사용자 흐름을 기준으로 자동화를 설계
- https://www.saucedemo.com/ 접속
- 로그인 테스트 (정상/비정상/경계값)  3개 작성
- 장바구니 흐름 테스트
- 요구사항 기반 테스트 케이스 설계
- 자동화 구조 설계 설명 (Playwright + POM 구조 적용)
- 실행 결과 리포트 첨부
- 결함 리포트 작성

---

### 🔹 테스트 범위

#### A. 로그인

1. 정상 계정으로 로그인 성공
2. 잘못된 아이디로 로그인 실패
3. 잘못된 비밀번호로 로그인 실패
4. 빈값 입력 (아이디/비밀번호 둘 다)
5. locked_out_user로 로그인 시도
6. 로그인 후 URL 정상 이동 확인

#### B. 장바구니

1. 상품 1개를 장바구니에 담기
2. 상품 여러개 담기
3. 장바구니에서 삭제 후 카운트 확인
4. 장바구니 페이지에서 상품 정보 일치 여부
5. 로그인 없이 장바구니 접근 시도

#### C. 주문

1. 정상 정보 입력 후 주문 완료
2. 필수 입력값 누락 시 에러 메시지
3. 주문 완료 후 장바구니 초기화 확인
4. 주문 완료 페이지 문구 확인

### 🔹 자동화 수행 과정
