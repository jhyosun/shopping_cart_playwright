# Playwright 활용 이커머스 웹 서비스 자동화 테스트
![image.png](images/image.JPG)

## 1. 프로젝트 목표

> [SauceDemo](https://www.saucedemo.com/) 데모 쇼핑몰을 대상으로 사용자 핵심 기능인 로그인, 상품 장바구니, 체크아웃 흐름에 대한 자동화 테스트를 수행했다.
> 

---

### 진행방향성

- 로그인 테스트 (정상/비정상/경계값)  3개 작성
- 장바구니 흐름 테스트
- 요구사항 기반 테스트 케이스 설계
- 자동화 구조 설계 설명 (Playwright + POM 구조 적용)
- 실행 결과 리포트 첨부
- 결함 리포트 작성

## 2. 테스트 범위

#### A. 로그인

1. 정상 계정으로 로그인 성공
2. 잘못된 아이디로 로그인 실패
3. 잘못된 비밀번호로 로그인 실패
4. 빈값 입력 (아이디/비밀번호 둘 다)
5. locked_out_user로 로그인 시도
6. 로그인 후 URL 정상 이동 확인

#### B. 장바구니

1. 상품 1개를 장바구니에 담기
2. 상품 여러개 담기
3. 장바구니에서 삭제 후 카운트 확인
4. 장바구니 페이지에서 상품 정보 일치 여부
5. 로그인 없이 장바구니 접근 시도

#### C. 주문

1. 정상 정보 입력 후 주문 완료
2. 필수 입력값 누락 시 에러 메시지
3. 주문 완료 후 장바구니 초기화 확인
4. 주문 완료 페이지 문구 확인

---

### 3. 테스트 케이스

10개정도생각

### 4. 테스트 자동화 코드

**로그인 기능 자동화**

- 정상 로그인
- 잘못된 아이디
- 잘못된 비밀번호
- 빈값 입력

**이커머스 장바구니**

- 상품 담기
- 수량 변경
- 삭제

### 5. 결함리포트

SauceDemo는 의도적으로 버그가 심어져 있음. **performance_glitch_user, problem_user** 계정으로 테스트하면 실제 결함. 이걸 결함리포트로 작성

- 로그인정보
    
    username: standard_user
    password: secret_sauce
    

### 진행할 테스트

- 로그인 성공/실패
- 상품 리스트 확인
- 장바구니 담기
- 장바구니 삭제
- 주문(Checkout)
